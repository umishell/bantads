================================================================================
FRAGMENTO COMUM — todos os prompts UI (Chrome MCP)
================================================================================
Inclua mentalmente estas regras em TODA sessão Agent (modo Agent obrigatório).

## Ferramenta
Servidor MCP: **cursor-ide-browser** (Chrome embutido).
URL base: **http://localhost/** (gateway + Angular + /api).

## Regras anti-freeze (OBRIGATÓRIO)
1. **Uma ação por vez** — nunca paralelizar `browser_navigate`, `browser_click`, `browser_fill`.
2. Ciclo por passo: **navigate → browser_lock → snapshot → ação → wait → snapshot → network (se necessário)**.
3. Após cada clique em botão Angular, **aguarde 2–5s** (`browser_wait_for`) antes do próximo snapshot.
4. Ao **terminar a sessão**: `browser_lock` com `action: unlock`.
5. Sagas: poll com waits de 5–10s, máximo **120s** total; só então marcar FAIL.
6. **Não** interromper a sessão no primeiro FAIL; registrar e continuar.

## Exceções permitidas (fora da UI)
- `GET http://localhost/api/integration/reboot` — reset seed (HTTP 200)
- Abrir **MailHog** em http://localhost:8025 (sessão saga apenas)
- `browser_network_requests` / `browser_console_messages` para evidência

## Proibido
- Swagger :808x, Postman ou microsserviços diretos como substituto da UI
- Várias navegações/cliques simultâneos na mesma resposta do agente

## Pré-requisitos
1. `docker compose ps` — stack running
2. Reboot: `GET http://localhost/api/integration/reboot` → 200
3. Senha seed: **tads**
